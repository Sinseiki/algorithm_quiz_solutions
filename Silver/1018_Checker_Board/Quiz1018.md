https://www.acmicpc.net/problem/1018

- Problem type: Brute-force / Implementation

- Goal: Find the minimum number of repaint operations needed to make some 8×8 part of the board a valid checkerboard.

- Conditions:
  - The board is given as rows of characters, where 'W' means white and 'B' means black.
  - A valid checkerboard alternates colors horizontally and vertically.
  - The top-left cell of a valid 8×8 checkerboard can start with either 'W' or 'B'.

- Input:
  - The first line contains two integers, height and width, representing the dimensions of the board.
  - The following lines describe the board, one row per line.

- Output
  - Print the minimum number of repaint operations required.

---
<details>
  <summary>- Solution approach:</summary>
  <pre>
  1. Read the dimensions of the board and the board itself from input.
  2. Define the two valid 8x8 checkerboard patterns: one starting with 'W' and the other starting with 'B'.
  3. Choose one possible 8×8 section by fixing its top-left position.
  4. For each row in that section, examine the 8 cells one by one.
  5. Compare each cell with the expected cell in the current checkerboard pattern.
  6. Count how many cells differ within that 8×8 section.
  7. Repeat the same process for the other checkerboard type.
  8. Apply this comparison to every possible 8×8 section and keep the minimum repaint count.
  9. Print the result.
  </pre>
</details>
